System Architecture

A detailed Staff-Level Data Scientist perspective on the Data, ML, and GenAI infrastructure powering the Chicago Crime platform.

BigQuery Data Platform

The foundation of the Chicago Crime Application relies on a robust ingestion and transformation layer orchestrated by Apache Airflow. We treat BigQuery as our warehouse and semantic layer.

Staff Insight: By decoupling ingestion from transformation, we follow the ELT paradigm. Raw data lands in GCS seamlessly, loads into BQ via external tables, and dbt handles the complex dimensional modeling and DAG-based materializations sequentially.

Core Components

  • Airflow Sensors/Operators: Daily operators ingest the official City portal endpoints.
  • Cloud Storage: Acts as the indestructible bronze/raw staging layer.
  • BigQuery & dbt: Handles SCD Type 2 logic, dimensional modeling, and creates materialized views for downstream ML consumptions.

Batch ML Pipeline

We predict future safety scores dynamically per community area. We pull aggregated historical features from standard BQ views, train a LightGBM model, and push high-confidence forecasts back into a transactional store (Postgres) to serve our FastAPI backend.

Staff Insight: Feature engineering is explicitly pushed down to the data warehouse layer via dbt. We let BigQuery's distributed compute handle the heavy aggregations, keeping the Python ML code strictly focused on algorithm training, hyperparameter tuning, and MLflow artifact tracking.

Core Components

  • Gold ML Views: Aggregated features (rolling averages, seasonality) derived from BigQuery.
  • LightGBM: Chosen for its superior handling of tabular data and speed, outperforming DNNs for this spatial-temporal regression task.
  • MLflow: Artifact store keeping track of parameters, metrics, and persisting our frozen models.
  • Postgres Sync: Predictions are synchronized into studio_area_snapshot for ultra-low latency reads by the UI.

Agentic RAG using LangGraph

The conversational capability is powered by an advanced Agentic RAG architecture powered by LangGraph, routing queries dynamically. It integrates standard Web Search tools and a Neo4j Model Context Protocol (MCP) server for deep, unconstrained graph querying.

Staff Insight: This isn't basic semantic search. By utilizing Mistral with LangGraph and MCP, we empower the LLM to autonomously write Cypher queries, validate syntax guardrails, and execute them safely against Neo4j. The routing agent (smaller Mistral model) saves cost/latency on simple tasks, while complex analytical asks invoke the larger Mistral model via LangGraph events.

Core Components

  • LangGraph Streaming: Real-time websocket streaming yields thinking, tool calls, and tokens sequentially.
  • Neo4j MCP: We expose the Graph Database via Anthropic's Model Context Protocol, ensuring structured, standardized tool-use interfaces for the LLM.
  • Mistral Router: A fast 20B LLM processes unstructured user asks and determines complexity.
  • Langsearch: Real-time fallback to current web context to handle data missing in the graph repo.
